# Roadmap Hero — обучающая платформа по backend-разработке

Личная обучающая платформа: роадмап + полноценный курс из **6 модулей и 36 уроков**
с теорией, примерами кода и практическими заданиями (квизы и код-задачи),
плюс **сборник задач к каждому модулю** (~60 задач: от лёгких к сложным,
с проверкой и пояснениями). Три темы оформления: чёрная, тёмная и светлая
(переключатель в шапке).

## Запуск

```bash
npm install
npm run dev      # http://localhost:3000
```

Опционально — backend для аккаунтов и синхронизации прогресса
(во втором терминале; без него платформа работает в гостевом режиме):

```bash
cd server && npm install && cd ..
npm run dev:server   # API на http://localhost:3001
```

Прочие команды:

```bash
npm run build    # typecheck + production-сборка в dist/
npm run preview  # локальный просмотр production-сборки
npm run lint     # ESLint
```

## Структура платформы

| Маршрут | Страница |
| --- | --- |
| `/` | Лендинг (роадмап, методы обучения, чек-лист задач) |
| `/courses` | Каталог модулей с общим прогрессом |
| `/courses/:course` | Обзор модуля со списком уроков |
| `/courses/:course/problems` | Сборник задач модуля (фильтр по сложности) |
| `/courses/:course/:lesson` | Урок: теория, код, квизы и задания |
| `/auth` | Вход и регистрация (если доступен API) |

Прогресс (пройденные уроки, упражнения, задачи сборников и чек-лист
с лендинга) хранится в `localStorage` браузера.
Если запущен backend (`server/`) и выполнен вход, прогресс дополнительно
синхронизируется с сервером и доступен с любого устройства; при входе
локальный и серверный прогресс объединяются. Устройство backend'а подробно
разобрано в [server/README.md](server/README.md).

## Деплой на Vercel

Проект готов к публикации «как есть»: подключите GitHub-репозиторий к Vercel —
пресет **Vite** подхватится автоматически (`npm run build` → `dist/`),
`vercel.json` уже настраивает SPA-маршрутизацию.

На Vercel деплоится **только фронтенд**. Приложение само определяет, что API
недоступен, прячет вход и работает в гостевом режиме — прогресс хранится
в `localStorage`. Чтобы включить аккаунты на проде, задеплойте `server/`
на любой Node-хостинг (Render, Railway, Fly.io — бесплатных тарифов
достаточно) и проксируйте на него `/api` через `rewrites` в `vercel.json`:

```json
{ "source": "/api/(.*)", "destination": "https://ваш-api.example.com/api/$1" }
```

(эту строку нужно добавить ПЕРВОЙ в массив `rewrites`).

## Как устроен контент

Весь учебный контент — типизированные данные в `src/data/courses/*.ts`
(схема — `src/types/course.ts`):

- **Блоки урока**: `heading`, `text`, `code` (с подсветкой Prism), `list`,
  `note` (info/tip/warning), `table`. В текстах поддерживается `**жирный**`
  и `` `код` `` инлайн-синтаксис.
- **Упражнения**: `quiz` (4 варианта, объяснение после правильного ответа) и
  `code` (задача с заготовкой, постепенными подсказками и эталонным решением).
- **Сборники задач**: `src/data/problems/<модуль>-1.ts` и `-2.ts` — те же
  типы упражнений плюс поле `difficulty: 'easy' | 'medium' | 'hard'`.

Чтобы добавить урок — допишите объект в массив `lessons` соответствующего
модуля; страницы, навигация и прогресс подхватят его автоматически.
Чтобы добавить задачу в сборник — допишите объект в `problems` любой из
частей (id должен быть уникальным и никогда не меняться: на него завязан
сохранённый прогресс).

## Темы оформления

Три темы: **чёрная** (по умолчанию), **тёмная** (графит) и **светлая**.
Палитра — CSS-переменные `--t-*` в `src/index.css`, семантические классы
(`bg-base`, `text-fg`, `border-line`, `text-brand`…) — в `tailwind.config.js`.
Выбор темы сохраняется в браузере (next-themes). WebGL-сцена лендинга и
панели кода намеренно остаются тёмными во всех темах.
